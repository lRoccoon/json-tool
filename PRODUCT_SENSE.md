# PRODUCT_SENSE.md

## Target Users

面向需要快速理解、清洗或检查 JSON 数据的开发者:

- 排查线上日志时,把一段被多次转义、夹带 Unicode 编码的 JSON 字符串恢复成可读结构。
- 在编写或调试接口时,验证 payload 形状、找特定字段、对比不同请求的差异。
- 处理嵌套 JSON(字符串里又是 JSON)、超长字段、含 markdown 的字段等不规则数据。

典型用户具备基础 JSON 知识,但不希望被复杂工具拖慢——他们要的是"粘上去就能看清楚"。

## Core Scenarios

1. **粘贴 → 解析 → 浏览**:从日志/抓包里复制一段 JSON,粘到 source pane,右侧立刻渲染可逐层展开的树。
2. **转义清洗**:输入是被多次转义的字符串(`\\\"`、`\\u4e2d`...),通过 Toolbar 一键 escape / unescape / Unicode⇄中文,直到拿到干净的 JSON。
3. **嵌套钻取**:某个字符串字段本身也是 JSON,在树视图上展开为子树继续浏览,不需要复制到新窗口。
4. **历史回看**:浏览了多份不同的 payload 后,从 History Panel 恢复任意一条早期快照。
5. **分享**:把当前内容上传到自托管的文本存储,得到一个 `#share=...` 链接,发给同事在他自己的浏览器里打开。
6. **长文本/Markdown 阅读**:遇到长字段或包含 markdown 的字符串,在 Value Popup 里切换 wrap、unescape、markdown 模式阅读。

## Value Judgments

- **可读 > 功能多**:树视图的可读性是第一优先级;不为了功能完整堆叠选项,让默认状态保持安静。
- **本地 > 远端**:能在本地完成的事情(解析、转义、历史)绝不上传;远端能力(分享)必须用户显式触发,且端点可配置。
- **延展性 > 立即**:面对超大 JSON,先给用户控制权(逐层展开、按需打开 popup),而不是一次渲染所有节点。
- **可靠 > 智能**:解析错误直接显示原始消息,不做"自动修复"猜测。

## Non-Goals

- 不做 JSON 编辑器:不支持在树视图上修改字段、增删节点(只允许"删除某子树后查看结果")。
- 不做账户系统、不存云端历史。
- 不做团队协作(实时编辑、评论)。
- 不做 schema 校验或 diff——这些有更专业的工具。
- 不做埋点/统计。

## Acceptance Lens

判断一个能力"做对了"的口径:

- 用户粘贴一段陌生 JSON,**5 秒内**能看到结构化的树。
- 处理百兆级粘贴时,UI 不冻结超过 1 秒(逐层展开、按需渲染应能挡住此类输入)。
- 默认 UI 状态下不应有任何外部网络请求;打开 DevTools Network 面板应为空。
- 任何"潜在副作用"操作(清空历史、上传分享)必须有明显的确认或显式入口。
